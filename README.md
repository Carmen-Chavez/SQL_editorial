# Análisis de Base de Datos de Libros
**Contexto:** Durante la pandemia, más personas se quedaron en casa leyendo libros, lo que atrajo la atención de startups que desarrollan aplicaciones para amantes de la lectura.
## Objetivo
Analizar una base de datos de un servicio competidor para generar una propuesta de valor para un nuevo producto.

- Las 5 Consultas SQL 
1. Libros recientes:

- Encuentra libros publicados después del 1 de enero de 2000
2. Análisis de popularidad:

- Número de reseñas y calificación promedio por libro
3. Editorial más productiva:

- Editorial con más libros publicados (>50 páginas)
4. Autor mejor calificado:

- Autor con mayor calificación promedio (libros con ≥50 calificaciones)
5. Usuarios activos:

- Promedio de reseñas de usuarios que calificaron >50 libros

## Habilidades a desarrollar
- Consultas SQL complejas con JOINs
- Funciones de agregación (COUNT, AVG, etc.)
- Filtrado avanzado con WHERE y HAVING
- Conexión a bases de datos con Python
- Análisis de datos para decisiones de negocio

## Metodología de Trabajo
Para cada consulta:

- Se describio el objetivo del análisis
- Escribio la consulta SQL
- Generaó resultados en el notebook
- Describió conclusiones de cada tarea
- Conexión a la Base de Datos
- Se uso PostgreSQL con credenciales específicas y pandas para ejecutar las consultas.
